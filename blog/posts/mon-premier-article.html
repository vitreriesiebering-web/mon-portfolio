Mon premier article

27 mars 2026

Quand j'ai décidé de me lancer dans le développement, je ne savais pas par où commencer. Il y a tellement de langages, de frameworks, d'outils — c'est facile de se perdre. Alors j'ai choisi l'approche la plus directe : construire quelque chose de concret.

Pourquoi un portfolio ?

Un portfolio, c'est le projet parfait pour débuter. Il combine les bases du web — HTML pour la structure, CSS pour le style, JavaScript pour l'interactivité — dans un projet qui a un sens personnel. Ce n'est pas un exercice abstrait : c'est ton site, ta vitrine.

J'ai choisi de le construire sans framework, sans outil de build, juste avec les fondamentaux. L'idée était d'apprendre les bases avant de monter en complexité.

Construire avec Claude Code

Ce qui a rendu l'expérience unique, c'est l'utilisation de Claude Code comme partenaire de développement. Plutôt que de suivre un tutoriel pas à pas, j'ai pu :

  • Décrire ce que je voulais en français, et obtenir du code fonctionnel
  • Comprendre chaque décision technique en posant des questions
  • Itérer rapidement sur le design et les fonctionnalités
  • Apprendre les bonnes pratiques (BEM, variables CSS, accessibilité) en contexte
Le meilleur moyen d'apprendre à coder, c'est de coder. Mais avoir un mentor disponible 24h/24 change tout.

Ce que j'ai appris

En construisant ce portfolio, j'ai découvert les variables CSS, le responsive design avec les media queries, les animations avec IntersectionObserver, et la gestion de thème dark/light avec localStorage. Chaque fonctionnalité ajoutée m'a appris quelque chose de nouveau.

La prochaine étape ? Continuer à construire, article après article, projet après projet.
