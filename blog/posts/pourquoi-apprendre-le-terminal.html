Pourquoi apprendre le terminal

31 mars 2026

Avant de toucher au terminal, je faisais tout avec des interfaces graphiques. Créer un dossier ? Clic droit, nouveau dossier. Renommer un fichier ? Double-clic. Ça marche, mais c'est lent — et surtout, ça ne scale pas.

Le déclic

Mon déclic est venu avec Git. Impossible de vraiment comprendre Git sans passer par la ligne de commande. Les interfaces graphiques cachent trop de choses. Quand j'ai tapé mon premier git init suivi d'un git commit, j'ai compris ce qui se passait réellement sous le capot.

Les commandes qui changent tout

Pas besoin de tout apprendre d'un coup. Ces commandes couvrent déjà beaucoup de terrain :

  • cd, ls, mkdir — naviguer et organiser ses fichiers
  • git add, git commit, git push — versionner son code
  • python3 -m http.server — lancer un serveur local en une ligne
  • cat, grep — lire et chercher dans des fichiers

Le terminal et l'IA

Ce qui est intéressant aujourd'hui, c'est que le terminal est aussi l'interface naturelle pour travailler avec des outils d'IA comme Claude Code. On tape une instruction, on obtient un résultat. C'est direct, rapide, et ça force à formuler clairement ce qu'on veut.

Le terminal ne pardonne pas l'approximation, et c'est exactement ce qui en fait un bon professeur.

Conseils pour débuter

Ne pas essayer de tout mémoriser. Utiliser le terminal pour des tâches concrètes — créer un projet, commiter du code, lancer un serveur — et chercher les commandes au fur et à mesure. La mémoire musculaire vient avec la pratique, pas avec les flashcards.
