Les bases de CSS Grid

29 mars 2026

Quand j'ai commencé à vouloir organiser mes éléments sur une page, j'ai d'abord utilisé Flexbox pour tout. Ça marche bien pour des alignements simples, mais dès qu'on veut une vraie grille — comme des cartes de projets côte à côte — CSS Grid change la donne.

Le concept en une phrase

CSS Grid permet de découper un conteneur en lignes et colonnes, puis de placer les éléments enfants dans cette grille. C'est comme un tableau, mais en beaucoup plus flexible.

Le minimum pour démarrer

Trois propriétés suffisent pour créer une grille fonctionnelle :

  • display: grid — active le mode grille sur le conteneur
  • grid-template-columns — définit le nombre et la taille des colonnes
  • gap — espace entre les éléments

Par exemple, pour afficher trois cartes de projets côte à côte :

grid-template-columns: repeat(3, 1fr) crée trois colonnes de taille égale. C'est la ligne que j'utilise le plus souvent.

Grid vs Flexbox

La règle simple que j'applique : Flexbox pour les alignements en une dimension (une ligne ou une colonne), Grid pour les mises en page en deux dimensions (lignes et colonnes). Dans ce portfolio, les cartes de projets utilisent Grid, tandis que la navbar utilise Flexbox.

Ce que j'ai retenu

CSS Grid n'est pas compliqué une fois qu'on comprend le modèle mental : un conteneur divisé en cellules. Pas besoin de maîtriser toutes les propriétés au départ — grid-template-columns, gap et repeat() couvrent 80 % des cas.

Le meilleur conseil : ouvrir les DevTools du navigateur, activer l'affichage de la grille, et expérimenter.
